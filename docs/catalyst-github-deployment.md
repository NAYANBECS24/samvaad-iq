# Catalyst GitHub Deployment Plan

This repository is prepared for a Catalyst-first submission path. The app remains demo-safe with bundled synthetic data, while the deploy target is Catalyst Slate or Web Client Hosting for the React client and Catalyst Serverless Functions for the API.

## Why This Shape

Official Catalyst docs state that Pipelines can integrate with GitHub, GitLab, or Bitbucket and trigger automatically when changes are pushed to the linked repository. The same docs identify `catalyst-pipelines.yaml` as the workflow file executed by the pipeline.

Official Catalyst CLI docs also state that `catalyst.json` is required when deploying a whole app from the CLI or from an external source such as GitHub. Web Client Hosting requires a `client-package.json` in the client resource root, so the Vite build now copies `client/public/client-package.json` into `dist`.

## Repository Artifacts

- `catalyst.json`: maps the deployable client output to `dist` and deploys the `functions/api` target.
- `catalyst-pipelines.yaml`: installs dependencies, lints, builds, smoke-tests, installs Catalyst CLI, and runs `catalyst deploy`.
- `client/public/client-package.json`: copied into `dist` for Catalyst Web Client Hosting metadata.
- `data/catalyst-service-map.json`: shared service-readiness map used by docs, UI, and the API readiness endpoint.
- `functions/api/modules/catalystReadiness.js`: exposes the readiness map through `GET /api/catalyst/readiness`.

## GitHub To Catalyst Flow

1. Push this repository to GitHub:

```powershell
git add .
git commit -m "Prepare SAMVAAD-IQ for Catalyst deployment"
git push origin main
```

2. In Catalyst Console, create or open the project, then create a Catalyst Pipeline.

3. Choose GitHub as the integration source and link the repository and branch.

4. Configure these Catalyst Pipeline variables:

- `PROJECT_ID`: Catalyst project identifier accepted by the CLI deploy command.
- `CATALYST_ORG`: Catalyst organization value from the console.
- `CATALYST_TOKEN`: Catalyst CLI token generated by the owner account.
- `USER_NAME`: Docker Hub username for the pipeline image pull.
- `USER_PASSWORD`: Docker Hub password/token for the pipeline image pull.

5. Push to the linked branch. Catalyst should fetch the repo, run `catalyst-pipelines.yaml`, build `dist`, run smoke tests, and deploy.

## Slate Git Deployment Option

For a frontend-only Slate Git deployment, use these settings:

- Repository root: `samvaad-iq`
- Install command: `npm install`
- Build command: `npm run build`
- Output path: `dist`

The API still needs Catalyst Serverless Functions deployment for the full prototype.

## Local Validation Before Push

```powershell
npm run client:lint
npm run build
npm run api:smoke
```
